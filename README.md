# 🚀 Multi-Stack Dockerized Web Template

โปรเจกต์นี้เป็นคอลเลกชันของเว็บแอปพลิเคชันเทมเพลตแบบ Fullstack (MERN Stack) สำหรับใช้เป็นรากฐานในการพัฒนาแอปพลิเคชันขนาดเริ่มต้นและระดับกลาง โดยนำเสนอโครงสร้างสแต็กสไตล์สากลที่แยกจากกันอย่างเป็นระบบ และควบคุมการทำงานร่วมกันผ่าน Docker Compose มีให้เลือกใช้งาน 2 เวอร์ชันตามความชอบของนักพัฒนา:

1. **`vanilla-js`:** เวอร์ชันดั้งเดิม เขียนด้วย HTML, CSS และ JavaScript แท้โดยไม่พึ่งพาคอมโพเนนต์เฟรมเวิร์กใดๆ (เหมาะสำหรับความเบาตัวและง่ายต่อความเข้าใจพื้นฐาน)
2. **`vue-js`:** เวอร์ชันโมเดิร์น พัฒนาด้วย Vue.js 3 ผ่านการเชื่อมโยงระบบ Reactive Composition API (เหมาะสำหรับการพัฒนาในระดับสูงที่มีหน้าตาซับซ้อนขึ้น)

---

## 🏗️ โครงสร้างโฟลเดอร์หลัก (Workspace Structure)

```text
template-web/
  ├── README.md               # คู่มือการใช้งานภาพรวมของพื้นที่พัฒนานี้
  │
  ├── vanilla-js/             # [เทมเพลตสแต็กดั้งเดิม]
  │   ├── docker-compose.yml  # ไฟล์ตั้งค่าสำหรับ Nginx (Port 80) + Node (Port 3000) + MongoDB
  │   ├── .env                # เก็บตัวแปรสแต็ก (รหัสผ่าน DB)
  │   ├── nginx/              # Proxy Server และไฟล์ Config
  │   ├── backend/            # APIs (Node.js + Express + Mongoose)
  │   └── frontend/           # Static Client (HTML + CSS + Pure JS)
  │
  └── vue-js/                 # [เทมเพลตสแต็กโมเดิร์น]
      ├── docker-compose.yml  # ไฟล์ตั้งค่าสำหรับ Nginx (Port 8080) + Node (Port 3000) + MongoDB
      ├── .env                # เก็บตัวแปรสแต็ก
      ├── nginx/              # Proxy Server และไฟล์ Config
      ├── backend/            # APIs
      └── frontend/           # Reactive Client (HTML + CSS + Vue 3 Composition API)
```

## ⚙️ ข้อมูลการรันของแต่ละสแต็ก (Services Port Configurations)

เพื่อช่วยให้นักพัฒนาสามารถทดลองรันและทำงานกับทั้งสองสแต็กพร้อมกันได้โดยไม่มีการขัดข้องทางระบบ (Port Collisions) พอร์ตการรันของสองฝั่งจึงถูกปรับแต่งแยกออกจากกันดังนี้:

| บริการ / สแต็ก | เวอร์ชัน `vanilla-js` (พอร์ตบนเครื่องโฮสต์) | เวอร์ชัน `vue-js` (พอร์ตบนเครื่องโฮสต์) |
|---|---|---|
| **Web App (Nginx)** | 🟢 **`80`** (เข้าผ่าน `http://localhost`) | 🔵 **`8080`** (เข้าผ่าน `http://localhost:8080`) |
| **MongoDB Database** | 🟢 **`27017`** | 🔵 **`27018`** |
| **Internal Backend API** | พอร์ต 3000 (ภายในเครือข่าย Docker) | พอร์ต 3000 (ภายในเครือข่าย Docker) |

## 🏃 วิธีเริ่มใช้งานระบบ (How to Run)

ก่อนทำงาน ให้ตรวจสอบว่าเครื่องคอมพิวเตอร์ของคุณติดตั้งและสตาร์ท **Docker / Docker Desktop** เรียบร้อยแล้ว จากนั้นดำเนินการดังนี้:

### 1. เข้าไปที่โฟลเดอร์สแต็กที่ต้องการรัน
เปิด Terminal ในโฟลเดอร์หลักของโปรเจกต์นี้ จากนั้นเลือก:

**รันเวอร์ชัน Vanilla JS:**
```bash
cd vanilla-js
docker-compose up --build -d
```

**รันเวอร์ชัน Vue JS:**
```bash
cd vue-js
docker-compose up --build -d
```

*ระบบจะดำเนินการดาวน์โหลด Image บิลด์โค้ด และสั่งรัน 3 คอนเทนเนอร์ (Web server, Node API, MongoDB) ในพื้นหลังโดยอัตโนมัติ*

### 2. ทดลองเปิดหน้าเว็บ
* **Vanilla JS Stack:** ไปที่ **`http://localhost`**
* **Vue.js Stack:** ไปที่ **`http://localhost:8080`**

### 3. ตรวจสอบสถานะการเชื่อมต่อ
หน้าเว็บทั้งสองเวอร์ชันจะมีระบบ **Service Status Widget** ที่จะคอยทำ Health Check และสื่อสารผลแบบเรียลไทม์ (เมื่อทุกคอนเทนเนอร์ทำงานสมบูรณ์ สเตตัสทั้ง 3 ตัวจะต้องเป็นจุดสีเขียว 🟢 ทั้งหมด)

## 🛑 คำสั่ง Docker ที่สำคัญในการดูแลระบบ
ขณะอยู่ในโฟลเดอร์สแต็ก (`vanilla-js` หรือ `vue-js`) สามารถรันคำสั่งเหล่านี้ได้ใน Terminal:

* **หยุดการทำงาน (และเก็บรักษาข้อมูลในฐานข้อมูลไว้):**
  ```bash
  docker-compose down
  ```
* **หยุดการทำงานพร้อมสั่ง Reset ฐานข้อมูลทั้งหมดให้ว่างเปล่า:**
  ```bash
  docker-compose down -v
  ```
* **เรียกดู logs การทำงานของทุกระบบแบบเรียลไทม์:**
  ```bash
  docker-compose logs -f
  ```
